# whisper-script - 동영상 자막 추출기

> 경로: `C:\Users\ganna\Downloads\whisper-tool`
> 실행: `python app.py` → http://127.0.0.1:7860

## 개요

| 항목 | 내용 |
|------|------|
| **유형** | 스크립트 추출기 (동영상 → 자막/텍스트) |
| **한줄 설명** | 동영상을 드래그&드롭하면 자막(SRT/VTT) + 텍스트(TXT) + AI 요약용 클린 텍스트를 자동 생성 |
| **타겟** | 강의 녹화, 회의 녹음 등 음성→텍스트 변환이 필요한 경우 |
| **테마** | 라이트 (Toss 스타일) |
| **폰트** | Pretendard Variable (CDN) |

## 기술 스택

```
Python + Gradio + faster-whisper + CUDA fp16 + imageio-ffmpeg
```

| 기술 | 용도 |
|------|------|
| **Gradio** | 웹 UI (드래그&드롭 + 진행률 + 실시간 미리보기) |
| **faster-whisper** | CTranslate2 기반 Whisper 추론 (large-v3 모델) |
| **CUDA fp16** | GPU 가속 (CPU 대비 5~10배 빠름) |
| **imageio-ffmpeg** | FFmpeg 내장 (별도 설치 불필요) |
| **torch** | GPU 감지 + 모델 로딩 |

## 핵심 기능

| 기능 | 설명 |
|------|------|
| **배치 처리** | 여러 동영상 동시 업로드, 순차 처리 + 큐 표시 |
| **다중 포맷** | TXT (순수 텍스트) + SRT (자막) + VTT (웹 자막) 선택 출력 |
| **AI 요약용 전처리** | `_clean/` 폴더에 필러 제거 + 중복 제거된 텍스트 자동 생성 |
| **실시간 미리보기** | 추출 중 트랜스크립트 실시간 표시 + 파일별 구분 헤더 |
| **ZIP 다운로드** | 완료 후 전체 파일 ZIP 패키지 |
| **모델 선택** | tiny ~ large-v3 (5단계, 품질 vs 속도 선택) |
| **다국어** | 한국어/영어/일본어/중국어/자동감지 |

## 출력 구조

```
~/Documents/Whisper Output/
├── 영상1.txt          ← 순수 텍스트
├── 영상1.srt          ← SRT 자막
├── _clean/
│   └── 영상1.txt      ← AI 요약용 (필러/중복 제거)
```

## AI 요약 최적화 (`clean_text` / `clean_srt`)

### 왜 전처리가 필요한가

| 구분 | SRT 원본 | _clean 텍스트 | 절감율 |
|------|---------|--------------|--------|
| 타임스탬프/번호 | 있음 | 제거 | -40% |
| 필러 (음, 어, 네, 그쵸 등) | 있음 | 제거 | -15~20% |
| 중복 문장 | 있음 | 제거 | -10% |
| **총 토큰** | **100%** | **~35%** | **-65%** |

### 제거되는 필러 목록

```python
음, 어, 에, 아, 자, 네, 뭐, 이제, 그래서요, 있잖아요,
그쵸, 그렇죠, 그러니까, 아니, 예, 응, 맞아요, 그래요,
알겠습니다, 하여튼, 어쨌든, 뭐냐면, 뭐라 그러냐, 그니까
```

### AI 요약 비용 최적화

| 방식 | 토큰 | 비용 |
|------|------|------|
| SRT 원본 → Sonnet | 100% | 100% (기준) |
| `_clean` → Sonnet | ~35% | 35% |
| `_clean` → Haiku | ~35% | **~3.5%** |

> Claude Code에서 요약할 때: `_clean/` 파일을 사용하고, Task 에이전트에 `model: "haiku"` 지정

## 핵심 삽질 방지

- FFmpeg 별도 설치 불필요 — `imageio-ffmpeg`가 내장 바이너리 제공
- Windows에서 `PATH` 충돌 시 `tempdir/whisper_ffmpeg/`에 복사해서 우선순위 확보
- `large-v3` 모델 첫 실행 시 ~3GB 다운로드 (이후 캐싱)
- GPU 메모리 부족 시 `medium` 모델로 변경 (품질 약간 하락, 속도 2배)
