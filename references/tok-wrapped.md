## Web Worker 구조

JS는 싱글 스레드라 무거운 파싱을 메인에서 돌리면 UI가 멈춤. Web Worker로 별도 스레드에서 처리.

```
메인 스레드 (app.js)              Worker 스레드 (worker.js)
─────────────────────            ─────────────────────────
UI 렌더링, 버튼, 애니메이션        파싱 + 통계 + MBTI + 관계유형
        │                                  │
        ├── postMessage(file) ──────────▶  │  파일 받아서 처리
        │  ◀──── postMessage(progress) ────┤  "32% ㅋㅋㅋ 세는 중..."
        │  ◀──── postMessage(complete) ────┤  결과 객체 전달
   카드 렌더링 시작                     worker.terminate()
```

| 특징 | 설명 |
|------|------|
| 별도 스레드 | 메인 UI 안 멈춤, 로딩 애니메이션 계속 동작 |
| postMessage 통신 | 메인↔Worker 간 메시지로만 데이터 주고받음 |
| DOM 접근 불가 | Worker에서 document, window 사용 못함 |
| 서버 불필요 | 브라우저 내장 기능, 백엔드 없이 동작 |

### 8계층 신호 상세 (1:1)

| 계층 | 수집 대상 | 판별 원리 |
|------|----------|----------|
| L1 존대법 | 메시지 끝 `/(요\|니다\|세요\|시오)/` | 양쪽 낮음=연인/친구, 양쪽 높음=직장, 비대칭=가족 |
| L2 호칭 | 문두 12자 `startsWith` 로맨틱/가족/직장/친구 + 하트이모지(주당빈도) | 자기야=연인, 엄마=가족, 님=직장, 야=친구 |
| L3 의례 | 굿모닝(6-9시)/굿나잇(22-2시)/식사/안전/뭐해/업무어 주당빈도 | 모닝+나잇=연인, 뭐해폭격=썸, 식사=가족, 업무어=직장 |
| L4 친밀도 | 사랑해(L5)>보고싶(L4)>힘들었어(L3)+신체접촉+질투 | **존재 신호**: 사랑해 1번=0.5점 기본 + 주당빈도 보너스 |
| L5 시간대 | 24h 분포 벡터 vs 유형별 이상 프로필 코사인 유사도 | 9-18시=직장, 전시간대+심야=연인, 식사시간 피크=가족 |
| L6 역학 | 메시지량/길이 균형, 2시간+ 장시간 세션 비율 | 균형+장시간=연인, 불균형=썸, 짧은 세션=직장 |
| L7 언어 | 비속어/돈 키워드/애정디스(바보야~) | 비속어=친구, 돈=가족, 바보야=연인, 전부없음=직장 |
| L8 진화 | 대화 3등분 → 초반↔후반 존댓말/친밀도 변화 | 존댓말↓+친밀도↑=썸→연인, 변화없음=직장 |

### 핵심 설계 포인트

1. **메시지 1회 순회 O(n)** — 수천 개 메시지를 한 번만 돌면서 8개 데이터 동시 수집
2. **존재 신호** — "사랑해"는 빈도보다 있느냐 없느냐가 중요 (1번만 있어도 LOVER L4에 0.5점)
3. **주당 빈도 정규화** — 대화량 많든 적든 공정 비교 (메시지 비율 대신 주당 횟수 사용)
4. **코사인 유사도** — 24h 분포 벡터와 이상 프로필(LOVER/WORK/FAMILY 등)의 방향 유사도로 시간 패턴 매칭
5. **교차 감쇠** — 모순 방지: 로맨틱 신호 0개 → LOVER ×0.6 / 로맨틱 신호 3개+ → WORK ×0.4
6. **인티머시 게이트** — L3 의례(굿모닝/나잇)만으로 LOVER 과대평가 방지, L4 친밀도 있어야 LOVER L3 풀점수
7. **호칭 오탐 방지** — `includes` 대신 `startsWith` 사용 (교촌 "허니"콤보 → 로맨틱 오탐 차단)

### 알고리즘 흐름

```javascript
// === 1:1 (computeRelationship) ===
// 1. 메시지 1회 순회 → 8계층 누적 변수 동시 수집
// 2. 각 계층 0~1 스코어 산출 → 유형별 가중치(위 표) 곱해서 합산 → 0~100
// 3. 교차 감쇠 적용 (LOVER↔WORK 모순 방지)
// 4. 최고 점수 유형 = 판정 결과

// === 그룹 (computeGroupRelationship) ===
// 메시지 1회 순회 → 5개 신호(존대법비율, 호칭, 의례, 시간대 코사인유사도, 언어) 수집
// 3종(FRIEND/FAMILY/WORK) 가중합산 → 0~100 → 최고 점수 = 판정
// 그룹은 LOVER/FLIRT 제외 (쌍별 분석 불가)
```
