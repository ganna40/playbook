# pong - 퐁퐁 측정기

> URL: https://pong.pearsoninsight.com
> GitHub: https://github.com/ganna40/pong

## 개요

| 항목 | 내용 |
|------|------|
| **유형** | O/X 자가진단 테스트 |
| **한줄 설명** | 36개 O/X 질문 → 점수 → 그리스문자 등급 (시그마~오메가) |
| **타겟** | 결혼/연애 관련 바이럴 |
| **테마** | 다크 (보라 악센트) |
| **폰트** | 시스템 폰트 |

## 모듈 조합

```
BASE + DATA + SCREEN + QUIZ + GRADE + SHARE + GA + STYLE-DARK
```

## 특수 기능

| 기능 | 설명 |
|------|------|
| O/X 2지선다 | 예(O)/아니오(X) 단순 선택 |
| 가중 점수 (bs) | 질문별 기본 점수 1~4점 |
| 킬러 문항 | killer=true면 콤보 트리거 |
| 콤보 시스템 | 특정 질문 조합 시 보너스 점수 |
| 상관관계 | 한 질문의 O가 다른 질문 점수 증폭 |
| 성별 분기 | 남/여 질문셋 + 결과 메시지 다름 |
| 카테고리 피드백 | 카테고리별 임계값 초과 시 경고 |

## 점수 계산 흐름

```
기본 점수 합산
  + 콤보 보너스 (특정 질문 조합)
  + 상관관계 보너스 (연쇄 효과)
  + 킬러 문항 가중치
  = 최종 점수 → 등급 판정
```

## 등급 체계

| 등급 | 점수 | 이름 | 리스크 |
|------|------|------|--------|
| Σ | 0~8 | 시그마 | 없음 |
| Α | 9~20 | 알파 | 매우 낮음 |
| Β | 21~40 | 베타 | 낮음~중간 |
| Γ | 41~60 | 감마 | 중간 |
| Δ | 61~80 | 델타 | 높음 |
| Ω | 81~100 | 오메가 | 매우 높음 |

## AI에게 비슷한 거 만들게 하려면

```
playbook의 pong 레퍼런스를 보고
"번아웃 측정기"를 만들어줘.
[QUIZ] O/X 방식 + [GRADE] + [SHARE] + [STYLE-DARK]
- 30개 O/X 질문
- 콤보 시스템 포함
- 카테고리: 업무, 인간관계, 건강, 재정, 취미
- 등급: S/A/B/C/D/F
- data.json은 pong 구조 동일하게
```
