# 웹서비스 보안 실전 프롬프트 템플릿
> 새 서비스를 만들거나 기존 서비스를 점검할 때 바로 복붙해서 쓰는 1페이지용 프롬프트.

## 언제 쓰나

- 새 웹서비스를 만들기 직전
- 기존 서비스 보안 점검을 AI에게 바로 맡길 때
- 코드 수정과 사람 수동 설정 항목을 분리하고 싶을 때

## 복붙용 프롬프트

```text
이 프로젝트에서 웹서비스 보안 점검과 수정 작업을 해줘.

중요:
- 먼저 "AI가 바로 처리할 수 있는 항목"과 "사람이 직접 설정해야 하는 항목"을 구분해줘.
- AI가 처리할 수 있는 항목은 바로 코드/설정/문서 수정까지 진행해줘.
- 사람이 직접 해야 하는 항목은 실제 설정 위치, 설정 방법, 확인 방법까지 적어줘.
- 마지막에는 "지금 바로 사람이 해야 하는 작업"만 5~10줄로 따로 요약해줘.

점검 범위:
- 소스 탭: 클라이언트 번들에 secret, token, 내부 URL, admin 관련 하드코딩 값 노출 여부
- 네트워크 탭: 인증 누락, 민감정보 노출, CORS, 쿠키, 보안 헤더
- 콘솔 탭: 스택트레이스, 내부 경로, DB 정보 노출 여부
- /admin, /internal 페이지 접근 제어
- 관리자 API 인증 및 권한 검사
- 결제, 가격, 권한 검증의 서버 처리 여부
- DB 권한 정책, PostgreSQL RLS 또는 MySQL/MariaDB owner 필터
- Cloudflare, SSL, SSH, 포트, 백업, 모니터링

수정 원칙:
- 관리자 페이지와 관리자 API를 둘 다 막아라
- 환경변수와 secret은 클라이언트에 절대 노출하지 마라
- 프로덕션에서는 상세 에러를 숨기고 서버 로그에만 남겨라
- HttpOnly, Secure, SameSite 쿠키를 우선 검토하라
- 기존 기능을 최대한 안 깨는 최소 수정부터 적용하라

결과물 형식:
- AI가 처리한 항목
- 사람이 직접 해야 하는 항목
- 취약점 목록
- 심각도
- 수정한 파일 목록
- 변경 이유
- 직접 테스트할 체크리스트
- 지금 바로 사람이 해야 하는 작업
```

## 더 짧은 버전

```text
이 프로젝트 보안 점검해줘.

1. AI가 바로 처리할 항목 / 사람이 직접 해야 할 항목을 먼저 나눠줘.
2. AI가 할 수 있는 건 바로 수정해줘.
3. 사람 항목은 Cloudflare, SSL, env, MFA, 허용 IP, 배포 승인 중심으로 정리해줘.
4. 관리자 페이지, 관리자 API, 쿠키, CORS, 에러 노출, 비밀키 노출, DB 권한을 우선 점검해줘.
5. 마지막에 취약점 목록, 수정 파일, 테스트 체크리스트, 지금 당장 해야 할 일만 요약해줘.
```

## 사람이 미리 준비하면 좋은 것

- 운영 서버 접근 권한
- 도메인과 Cloudflare 접근 권한
- 실제 비밀값을 넣을 위치
- 관리자 허용 IP 정책
- 배포 승인 여부
